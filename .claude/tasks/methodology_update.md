# 预测方法论迭代优化任务

> 执行时间：北京时间每日 18:00（A股已收盘，今日预测结果已可核验）

## 第零步：更新代码仓库

```bash
git pull origin main || echo "git pull failed, continuing..."
```

方法论文件的固定路径为：data/methodology/prediction_methodology.md
后续预测agent每天早上会读取该文件作为预测依据，因此方法论必须保持抽象、普适、结构清晰。

## 第一步：读取今日预测

读取 data/predictions/ 目录中最新的 *_prediction.md 文件，提取：
- 今日预测的市场方向、幅度、重点板块
- 预测置信度
- 预设的检验标准

## 第二步：读取今日全天复盘

读取 data/daily_review/ 目录中最新的 *_full_review.md 文件，提取：
- 实际市场方向、涨跌幅
- 实际强势板块
- 市场超预期或低于预期的地方

## 第三步：对比评估今日预测得失

逐条对比预测与实际，明确判断：
- 方向预测：正确/错误（+1/-1分）
- 幅度预测：在区间内/偏大/偏小（+1/0/-1分）
- 板块预测：命中数/总预测数
- 今日预测总评：准确/基本准确/偏差较大
- 失误的具体原因分析（信号忽略？权重偏差？逻辑错误？）

## 第四步：读取现有方法论

读取 data/methodology/prediction_methodology.md（如文件不存在，则从零创建一个初始版本）。理解现有方法论的结构和核心规则。

## 第五步：更新方法论

根据今日得失，对方法论进行针对性修订。修订原则：
- **抽象性**：规则描述市场普遍规律，不绑定具体日期或个股
- **普适性**：适用于不同市场环境（牛/熊/震荡）
- **可操作性**：每条规则应能直接指导预测决策
- **权重意识**：不同信号的重要性应有明确优先级排序
- **案例锚定**：每条重要规则附1-2个历史验证案例（日期+结果），帮助后续agent理解规则的来源和边界

修订方式：
- 若今日预测准确：强化对应规则或提高其权重
- 若今日预测失误：分析原因，修正规则或降低失效信号的权重，补充反例案例
- 若发现新规律：新增规则条目

## 第六步：写回方法论文件

将更新后的完整方法论写回 data/methodology/prediction_methodology.md。

文件格式要求：

```
# A股预测方法论
> 最后更新：YYYY-MM-DD | 版本：vN.N

## 核心理念
[2-3句话概括预测框架的哲学基础]

## 一、信号优先级体系
### 1.1 高权重信号（必须考量）
- [信号名称]：[描述] → [预测意义] [案例]
### 1.2 中权重信号（参考考量）
- ...
### 1.3 低权重/失效信号（谨慎使用）
- ...

## 二、市场环境识别
- 趋势市特征与应对策略
- 震荡市特征与应对策略
- 转折信号识别

## 三、美股→A股传导规则
- [传导规律描述] [有效条件] [失效条件]

## 四、板块轮动规律
- [规律描述] [案例]

## 五、常见陷阱与反例
- [陷阱描述]：[历史反例]

## 六、修订日志
| 日期 | 修订内容 | 触发原因 |
|------|----------|----------|
| YYYY-MM-DD | [修订摘要] | [今日预测得失] |
```

## 第七步：输出总结

输出以下内容：
1. 今日预测得失评分（方向/幅度/板块）
2. 本次方法论修订的核心变化（1-3条）
3. 方法论当前版本号

## 最后步骤：提交并推送至GitHub

```bash
git add data/
git commit -m "auto: 方法论更新 $(date +%Y-%m-%d)" || echo "nothing to commit"
for i in 1 2 3; do git push origin main && break || { echo "retry $i"; sleep 10; }; done
git log --oneline -1
git status
```

- 成功：「✅ 分析结果已成功推送至GitHub」
- 失败：「❌ 推送失败（已重试3次），错误：[详细错误信息]」
