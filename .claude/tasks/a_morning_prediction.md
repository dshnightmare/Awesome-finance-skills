# A股早盘走势预测任务

> 执行时间：北京时间每日早上 9:00（A股尚未开盘）

## 第零步：更新代码仓库

```bash
git pull origin main || echo "git pull failed, continuing..."
```

## 第一步：读取预测方法论（最重要）

读取 data/methodology/prediction_methodology.md，完整理解其中的：
- 信号优先级体系（高/中/低权重信号）
- 市场环境识别规则
- 美股传导规则
- 板块轮动规律
- 常见陷阱

如果文件不存在，说明方法论尚未建立，用通用分析逻辑继续。

## 第二步：读取昨日A股复盘记录

读取 data/daily_review/ 目录中最新的 *_full_review.md 文件，提取：
- 主要指数涨跌情况
- 强弱板块
- 资金流向特征
- 市场情绪状态

如果文件不存在，说明未找到并继续。

## 第三步：读取昨天美股行情

读取 data/us_market_review/ 目录中最新的 *_us_market.md 文件，提取：
- 三大指数涨跌幅及情绪倾向
- 科技股、实业股等与A股相关板块表现
- VIX、美债、美元动向
- 重要宏观消息

如果文件不存在，说明未找到并继续。

## 第四步：获取最新财经资讯（可选）

如果仓库中有可用的新闻获取工具，尝试获取今日早盘前的最新财经动态。

## 第五步：生成今日预测报告

**严格按照方法论的信号优先级和规则进行分析**，对各信号进行系统评估后做出预测。预测内容应明确且可检验。

将预测报告保存到 data/predictions/ 目录，文件名格式：YYYY-MM-DD_prediction.md（使用今天实际日期）。

报告结构：

### 一、信息输入汇总
- 方法论版本：[v?.?]
- 昨日A股情况：[主要指标小结]
- 昨日美股情况：[主要指标小结]
- 关键信号列表：[按方法论权重排序]

### 二、今日A股预测
- **市场方向**：明确给出偏多/偏空/震荡，并说明置信度（高/中/低）
- **预期幅度**：上证指数波动区间预估
- **重点关注板块**：预期表现较强的板块（列出2-3个，附方法论依据）
- **驱动逻辑**：本次预测的主要依据是哪些方法论规则
- **风险因素**：可能导致预测失误的事项

### 三、预测得失检验标准
1. 市场方向是否正确（涨/跌/震荡）
2. 指数涨跌幅度是否在预估区间内
3. 重点板块是否有表现

完成后输出预测内容摘要。

## 最后步骤：提交并推送至GitHub

```bash
git add data/
git commit -m "auto: A股走势预测 $(date +%Y-%m-%d)" || echo "nothing to commit"
for i in 1 2 3; do git push origin main && break || { echo "retry $i"; sleep 10; }; done
git log --oneline -1
git status
```

- 成功：「✅ 分析结果已成功推送至GitHub」
- 失败：「❌ 推送失败（已重试3次），错误：[详细错误信息]」
