# A股全天行情复盘任务

> 执行时间：北京时间每日 16:00（全天交易 9:30-15:00 已结束）

## 第零步：更新代码仓库

```bash
git pull origin main || echo "git pull failed, continuing..."
```

## 第一步：读取前一天的预测记录

读取仓库 data/ 目录，找到最近一次的预测报告或信号文件，记录其中的主要预测观点（涨跌方向、重点板块、关键价位等）。如果找不到预测记录，说明未找到并继续执行。

## 第二步：获取今日A股全天行情

运行以下命令获取结构化行情数据：

```bash
/opt/miniconda3/bin/python3 .claude/skills/kol-market-review/scripts/fetch_market_data.py
```

如命令失败，手动通过接口获取主要指数数据，包括：
- 上证指数、深证成指、创业板指、科创50、中证500、中证1000的全天涨跌幅及成交量
- 全天涨幅、跌幅、平盘个股数量
- 主力资金流向和热点板块
- 北向资金全天净流入流出（如可获取）

## 第三步：7位KOL多视角复盘

读取 `.claude/skills/kol-market-review/references/kol_frameworks.md`，获取所有7位KOL的分析框架，逐一按各KOL方法论分析今日行情：

1. **么瑞与小A**（技术波段）：分析量能、K线形态、蝗虫过境信号，给出仓位建议
2. **MR Dang**（宏观解读）：识别今日核心事件，分析市场定价偏差，标注受影响板块
3. **寒武纪的鳄鱼**（周期×价值）：评估央企资源板块信号，判断周期位置，检查机会条件
4. **派大星皮皮**（趋势配置）：回答三问（牛/震/熊、增量/存量/减量、多空），评估进攻方向
5. **Deep Van**（全球产业链）：分析全球链条信号，评估多空布局，识别地缘变化影响
6. **白白胖胖0**（估值+右侧）：分析技术形态，检查止损信号，标注关注标的
7. **龙开**（宏观叙事→期货）：更新宏观叙事，分析商品/期货信号，检查机械止损

每个KOL分析须：仅使用该KOL自己关注的变量；结论量化（仓位用成数，概率用%）；不关注的维度标注"该KOL不关注此项"。

最后提炼7个视角的：共识（≥5/7一致的判断）、主要分歧、明日关键变量。

## 第四步：预测 vs 实际对比分析

将实际行情与第一步中的预测逐条对比：
- 预测准确的部分（✓）
- 预测偏差的部分及原因分析（✗）
- 整体预测命中率评估（方向、板块、幅度分别评分）

## 第五步：生成完整复盘报告

将报告保存到 data/daily_review/ 目录，文件名格式：YYYY-MM-DD_full_review.md（使用今天实际日期）。

报告结构：

### 一、全天行情概览
主要指数涨跌幅表、成交量、涨跌家数统计

### 二、前日预测回顾（如有）
预测要点列表

### 三、预测 vs 实际对比
对比表（方向/板块/幅度三维评分），偏差原因分析

### 四、7位KOL多视角复盘
逐一展示每位KOL的分析结论（按第三步框架）

**综合研判**：
- 共识（≥5/7一致）
- 主要分歧及分歧来源
- 明日关键变量（3条）
- 综合仓位建议范围

### 五、经验总结与预测模型优化建议
基于今日复盘，提出对预测方法论的改进建议（如有）

完成后输出报告摘要（各章节核心结论，不超过500字）。

## 最后步骤：提交并推送至GitHub

```bash
git add data/
git commit -m "auto: A股全天复盘 $(date +%Y-%m-%d)" || echo "nothing to commit"
for i in 1 2 3; do git push origin main && break || { echo "retry $i"; sleep 10; }; done
git log --oneline -1
git status
```

- 成功：「✅ 分析结果已成功推送至GitHub」
- 失败：「❌ 推送失败（已重试3次），错误：[详细错误信息]」
